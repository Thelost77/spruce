# Spruce vs Pine Playback Controls Audit

Generated by minmax3 audit comparing spruce's playback controls against pine's working reference.

## Critical Findings (ranked by impact)

### 1. Space (play/pause) does nothing
**Pine:** Space falls through to `player.Update` which toggles `m.Playing` and calls `TogglePauseCmd(mpv, m.Playing)`.
**Spruce:** `app/model.go:206-210` intercepts space, passes `m.playerState.Playing` (un-toggled) to `TogglePauseCmd` → `SetPause(!Playing)` = same state = no-op. Never calls `syncQueueScreen`.
**Fix:** Remove the app-level space case; let it fall through to the player handler (which toggles correctly), then syncQueueScreen.

### 2. Seek (l/h) does nothing; h navigates back instead
**Pine:** Seek is intercepted at app level (`app/model.go:842-847`) calling `handleSeek(±seekSeconds)` → `mpv.Seek`. Runs before screens see the key.
**Spruce:** No app-level seek. `HandlesKey` returns true for seek keys but `handleKey` has no seek case → key leaks into screen → `h` triggers back-nav.
**Fix:** Add app-level seek interception issuing `player.SeekCmd(mpv, ±cfg.Player.SeekSeconds)`. Remove `h` from back-nav matchers.

### 3. MPRIS SetRateMsg unhandled
**Pine:** Handles `SetRateMsg` → sets speed + `SetSpeedCmd` + MPRIS emit.
**Spruce:** `mpris/messages.go` defines `SetRateMsg`, `bridge.go` sends it, but `app/model.go` has no `case mpris.SetRateMsg`.
**Fix:** Add the case.

### 4. MPRIS property-change signals never emitted
**Pine:** Emits `mprisPlaybackCmd`/`mprisPositionCmd`/`mprisVolumeCmd` after state changes.
**Spruce:** `syncQueueScreen` mutates `m.mprisState` but never fires D-Bus signals. External clients (playerctl) show stale state.
**Fix:** Port pine's MPRIS signal emission helpers.

### 5. Sleep timer missing entirely
**Pine:** `S` cycles 0/15/30/45/60 min, schedules `SleepTimerExpiredMsg`, stops playback on expiry. Footer shows countdown.
**Spruce:** No sleep timer code at all. `player.Model.SleepRemaining` field exists (copied) but never set.
**Fix:** Port pine's `cycleSleepTimer` + `SleepTimerExpiredMsg` + `S` key + footer already renders it.

### 6. App-level keys hardcoded, ignore config
**Pine:** `app/keymap.go` defines `KeyMap`/`DefaultKeyMap(cfg)`, all keys use `key.Matches(msg, m.keys.X)`.
**Spruce:** `app/model.go:199-251` hardcodes `"q"`/`" "`/`"n"`/`"p"`/`"s"` as string literals. No `keymap.go` in app package. Config keybinds ignored for app-level keys.
**Fix:** Port pine's `app/keymap.go`, replace string switch with `key.Matches`.

### 7. HandlesKey lies about seek
**Spruce:** `player/model.go:104-112` returns true for SeekForward/SeekBack, but `handleKey:150-196` has no seek cases. App thinks player consumed the key; key silently no-ops.
**Fix:** Add seek cases to `handleKey` OR remove seek from HandlesKey and handle at app level (pine's approach).

### 8. Back-nav collides with seek (h)
**Spruce:** `app/model.go:463` and `library/model.go:63,193` treat `h` as back. Pine's Back binding is `esc`+`left` only.
**Fix:** Drop `h` from back-nav matchers everywhere.

### 9. No MPRIS/queue sync after volume & speed changes
**Spruce:** `player/model.go:176-192` (volume) and `:159-174` (speed) call `SetVolumeCmd`/`SetSpeedCmd` but never `syncQueueScreen`/MPRIS.
**Fix:** Sync after volume/speed changes.

### 10. No Help overlay (?)
**Pine:** `?` toggles full help overlay.
**Spruce:** No help key.
**Fix:** Optional — port pine's help overlay.

## What works correctly (no fix needed)
- Volume up/down (`]`/`[`) — routing correct, player handles it
- Speed up/down (`+`/`-`) — routing correct, player handles it
- Next/prev track (`n`/`p`) — works (different semantics: tracks vs chapters)
- Quit (`q`) — stops playback before quit, reports to Jellyfin
- Player footer rendering — `playerState.View()` called in render stack
- PlayerKeyMap — identical to pine's, config-driven

## Prioritized Fix Order
1. **Space** — remove app-level intercept, let player handle it + sync
2. **Seek** — add app-level `handleSeek` + remove `h` from back-nav
3. **MPRIS SetRate** — add missing case
4. **Keymap** — port `app/keymap.go`, use config bindings
5. **Sleep timer** — port from pine
6. **MPRIS signals** — port emission helpers
